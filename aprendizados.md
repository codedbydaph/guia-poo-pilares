# Aprendizados Obtidos no Projeto

Este documento registra as principais competências técnicas e conceituais desenvolvidas ao longo da criação deste guia prático de Programação Orientada a Objetos (POO).

## Competências Técnicas Desenvolvidas

* **Versionamento de Código Avançado com Git**: Prática consolidada de desenvolvimento utilizando fluxo de trabalho baseado em branches descritivas, evitando commits diretos na branch principal (`master`), mantendo o histórico organizado e isolado por entregas lógicas.
* **Documentação Técnica em Markdown**: Estruturação de material didático limpo, utilizando hierarquia de cabeçalhos, blocos de código com realce de sintaxe e listas para facilitar a legibilidade do usuário final.
* **Modelagem Teórica e Didática**: Exercitar a habilidade de traduzir conceitos complexos de engenharia de software em analogias simples do mundo real e desafios de código aplicáveis.

## Consolidação Conceitual de POO

A criação do guia permitiu reforçar a importância prática de cada pilar da POO em arquiteturas reais:
1. **Abstração**: Entendimento de como selecionar apenas dados relevantes para o contexto do sistema.
2. **Encapsulamento**: Proteção de dados sensíveis e controle de estado através de métodos seguros.
3. **Herança**: Redução de redundâncias e reutilização de lógica de código.
4. **Polimorfismo**: Criação de interfaces flexíveis onde diferentes classes filhas respondem ao mesmo método de maneiras específicas.
