# Guia Prático: Os 4 Pilares da Programação Orientada a Objetos (POO)

## Objetivo
Este guia tem como propósito desmistificar os conceitos fundamentais do paradigma Orientado a Objetos de forma sequencial, prática e direta. O objetivo é fazer com que o leitor compreenda como sair do pensamento procedural e passe a modelar problemas do mundo real utilizando os pilares de Abstração, Encapsulamento, Herança e Polimorfismo.

## Público-alvo
Estudantes de tecnologia e desenvolvedores iniciantes que já dominam a lógica de programação básica (como variáveis, estruturas condicionais, laços de repetição e funções), mas que estão tendo o primeiro contato prático com o paradigma de POO.

## Pré-requisitos
* Lógica de programação bem consolidada.
* Familiaridade básica com sintaxe de código (exemplos práticos demonstrados em Python).
* Git e GitHub instalados para clonar e navegar pelo repositório.

## Estrutura
O conhecimento está organizado de forma modular na pasta `docs/` e `exemplos/`:

* **`docs/01-introducao.md`**: Explicação sobre o paradigma, além da diferença conceitual e prática entre Classes e Objetos.
* **`docs/02-conceitos.md`**: Imersão teórica e técnica nos 4 pilares (Abstração, Encapsulamento, Herança e Polimorfismo).
* **`docs/03-exercicios.md`**: Desafios e problemas práticos para fixação do conteúdo.
* **`docs/04-referencias.md`**: Links, livros e documentações recomendadas para aprofundamento.
* **`exemplos/exemplo-basico.md`**: Um arquivo de código completo, unificando todos os conceitos aplicados em um cenário real.

## Como navegar
Para extrair o melhor aproveitamento deste guia, sugerimos seguir a ordem numérica estabelecida na pasta `docs/`. 
1. Comece lendo a introdução teórica.
2. Avance para o detalhamento dos conceitos.
3. Abra o arquivo contido na pasta `exemplos/` para ver como a teoria se transforma em código.
4. Tente resolver os exercícios propostos sem olhar a solução.
