# 04. Referências e Materiais de Aprofundamento

Para continuar evoluindo no entendimento da Programação Orientada a Objetos e boas práticas de arquitetura de software, recomendo os seguintes livros, documentações e artigos:

---

## 📚 Livros Recomendados

*   **Pense em Python** (Allen B. Downey)
    *   *Por que ler:* Excelente para iniciantes. Possui capítulos dedicados exclusivamente à modelagem de classes, objetos e métodos de forma muito didática.
*   **Python Fluente** (Luciano Ramalho)
    *   *Por que ler:* Essencial para quem quer entender como o Python manipula objetos por debaixo dos panos (métodos dunder, modelo de dados e herança múltipla).
*   **Engenharia de Software Clean Code: Habilidades Práticas para o Código Limpo** (Robert C. Martin)
    *   *Por que ler:* Aborda como estruturar classes de forma limpa, focando no princípio de responsabilidade única e coesão.

## 🌐 Documentação Oficial e Artigos

*   **Python Documentation - Classes**
    *   [docs.python.org/3/tutorial/classes.html](https://docs.python.org/3/tutorial/classes.html)
    *   A explicação oficial sobre o modelo de classes do Python, escopos e namespaces.
*   **Real Python - Object-Oriented Programming (OOP) in Python 3**
    *   Um dos melhores tutoriais práticos da comunidade para fixar conceitos de herança e propriedades.
